# 结构引擎 (Story Structure)

> 基于经典叙事框架的故事结构规划技能

## 功能描述

结构引擎技能允许作者选择经典的故事结构框架，AI 将根据框架的节拍定义和已有的故事简介，自动为每个节拍生成具体的故事内容。每个节拍包含名称、位置百分比、描述、情绪基调和创作指导，帮助作者快速搭建完整的故事骨架。

## 支持的框架（7 种）

| 框架 ID | 框架名称 | 节拍数 | 适用类型 |
|---------|----------|--------|----------|
| `three_act` | 三幕结构 | 8 | 通用型 |
| `hero_journey` | 英雄之旅 | 12 | 奇幻/玄幻/冒险 |
| `save_the_cat` | Save the Cat | 15 | 类型小说 |
| `five_act` | 故事圆环 | 8 | 网络连载/短篇 |
| `kishotenketsu` | 起承转合 | 4 | 东方叙事 |
| `seven_point` | 七点结构 | 7 | 通用 |
| `custom` | 自定义 | 自由 | 实验性写作 |

### 框架详细说明

#### 1. 三幕结构 (`three_act`) — 8 节拍

最经典的故事结构，将故事分为建置、对抗和解决三个部分，适用于几乎所有类型的叙事作品。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 建置 | 0% | 平静、日常 |
| 触发事件 | 10% | 惊讶、不安 |
| 第一转折点 | 25% | 紧张、期待 |
| 上升行动 | 37% | 波折、成长 |
| 中点 | 50% | 震撼、觉醒 |
| 第二转折点 | 75% | 紧迫、绝望 |
| 高潮 | 90% | 激烈、巅峰 |
| 结局 | 100% | 释然、满足 |

#### 2. 英雄之旅 (`hero_journey`) — 12 节拍

约瑟夫-坎贝尔提出的经典神话结构，描述英雄从平凡世界出发、经历试炼、获得蜕变并回归的完整旅程。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 平凡世界 | 0% | 平静、暗示 |
| 冒险召唤 | 8% | 诱惑、恐惧 |
| 拒绝召唤 | 15% | 犹豫、挣扎 |
| 遇见导师 | 20% | 启发、温暖 |
| 跨越第一道门槛 | 25% | 紧张、决绝 |
| 考验、盟友与敌人 | 35% | 好奇、警惕 |
| 接近深层洞穴 | 45% | 紧张、预感 |
| 严峻考验 | 50% | 绝望、重生 |
| 获得奖赏 | 58% | 喜悦、满足 |
| 返回之路 | 67% | 紧迫、不安 |
| 复活 | 80% | 极致紧张 |
| 携万灵药归来 | 100% | 圆满、升华 |

#### 3. Save the Cat (`save_the_cat`) — 15 节拍

布莱克-斯奈德提出的 15 个节拍的结构模板，精确到页面百分比，是类型小说和商业写作的实用工具。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 开场画面 | 0% | 定调 |
| 铺设主题 | 5% | 暗示 |
| 铺设设定 | 10% | 展示 |
| 触发事件 | 10% | 转折 |
| 辩论 | 20% | 犹豫 |
| 进入第二幕 | 25% | 决断 |
| B 故事 | 22% | 温暖 |
| 趣味与游戏 | 30% | 娱乐 |
| 中点 | 50% | 反转 |
| 坏人逼近 | 55% | 压迫 |
| 失去一切 | 75% | 绝望 |
| 灵魂暗夜 | 75% | 沉思 |
| 进入第三幕 | 80% | 觉醒 |
| 终局 | 85% | 高潮 |
| 终场画面 | 100% | 收束 |

#### 4. 故事圆环 (`five_act`) — 8 节拍

丹-哈蒙基于坎贝尔英雄之旅简化的 8 步结构，强调角色的欲望与变化，非常适合网络连载和短篇故事。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 你（处于舒适区） | 0% | 平静 |
| 需要（但有所渴望） | 12% | 渴望 |
| 出发（进入未知） | 25% | 紧张 |
| 寻找（适应新规则） | 37% | 困惑、学习 |
| 找到（获得想要的东西） | 50% | 短暂喜悦 |
| 代价（付出沉重代价） | 62% | 痛苦 |
| 回归（回到起点） | 75% | 沉淀 |
| 改变（完成蜕变） | 100% | 升华 |

#### 5. 起承转合 (`kishotenketsu`) — 4 节拍

源自东亚传统的四段式叙事结构，强调"转"的意外性而非冲突驱动，适合注重氛围和意境的作品。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 起（引入） | 0% | 宁静、铺陈 |
| 承（发展） | 33% | 温和、深入 |
| 转（转折） | 66% | 意外、震撼 |
| 合（收束） | 100% | 余韵、回味 |

#### 6. 七点结构 (`seven_point`) — 7 节拍

丹-威尔斯提出的简洁高效的结构方法，从结局开始反向规划，确保故事的每个转折都有因果逻辑。

| 节拍 | 位置 | 情绪基调 |
|------|------|----------|
| 钩子（Hook） | 0% | 对比 |
| 情节转折一（Plot Turn 1） | 25% | 推动 |
| 夹击一（Pinch 1） | 37% | 压迫 |
| 中点（Midpoint） | 50% | 觉醒 |
| 夹击二（Pinch 2） | 62% | 绝望 |
| 情节转折二（Plot Turn 2） | 75% | 顿悟 |
| 解决（Resolution） | 100% | 收束 |

#### 7. 自定义 (`custom`) — 自由节拍

不受任何框架限制，作者可以自由定义故事结构的节拍数量和内容，适合实验性写作和非线性叙事。

## 每个节拍包含的字段

```typescript
interface FrameworkBeat {
  name: string;        // 节拍名称
  position: number;    // 在故事中的位置百分比 (0-100)
  description: string; // 节拍描述模板
  emotion_tone: string; // 情绪基调
  guidance: string;    // 创作指导
}
```

> 类型定义来源：`src/lib/conception/frameworks.ts` — `FrameworkBeat` 接口

## AI 任务类型

`outline`

## API 端点

```
POST /api/conception/structure
```

### 请求示例

```json
{
  "framework": "hero_journey",
  "synopsis": "一个被放逐的王子踏上了寻找失落神器的旅程，途中他必须面对自己家族的黑暗秘密，并做出关乎整个王国命运的抉择。",
  "characters": [
    {
      "name": "亚瑟",
      "role": "protagonist",
      "want": "夺回王位",
      "need": "学会信任他人",
      "fear": "重蹈父亲的覆辙"
    }
  ]
}
```

### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `framework` | `StructureFramework` | 是 | 框架 ID |
| `synopsis` | `string` | 是 | 故事简介 |
| `characters` | `Character[]` | 否 | 已有角色列表 |

### 响应

返回 `text/stream` 流式响应，输出 JSON 数组，每个元素对应一个节拍的详细内容：

```json
[
  {
    "beatIndex": 0,
    "name": "平凡世界",
    "content": "该节拍的具体内容描述（3-5句话）",
    "characters": ["亚瑟"],
    "emotionTone": "平静、暗示",
    "keyEvents": ["亚瑟在边境村庄隐姓埋名", "村民谈论王国近况"]
  }
]
```

> API 实现文件：`src/app/api/conception/structure/route.ts`

## 前端组件

```
src/components/conception/structure-engine.tsx   — 结构引擎主面板
src/components/conception/beat-sheet-editor.tsx   — 节拍表编辑器
```

### 组件功能

- **结构引擎面板**：框架选择、一键生成、节拍概览
- **节拍表编辑器**：可视化编辑每个节拍的内容，支持拖拽排序

## 数据源

```
src/lib/conception/frameworks.ts
```

该文件导出：
- `FRAMEWORKS`：所有框架的完整定义（`Record<StructureFramework, StoryFramework>`）
- `getFrameworkList()`：获取框架列表
- `getFramework(id)`：根据 ID 获取单个框架

## 使用示例

**场景**：作者选择英雄之旅框架规划奇幻小说

1. 在结构引擎面板中选择 `hero_journey` 框架
2. 填写故事简介
3. 添加已有角色信息
4. 点击"生成结构"
5. AI 为 12 个节拍分别生成具体内容
6. 在节拍表编辑器中查看和调整每个节拍
7. 确认后保存到项目结构
